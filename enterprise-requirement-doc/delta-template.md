# 中间稿模板：需求新增点 / 优化点 / 改造点

本文件定义 `01-需求新增优化改造点.md` 的标准结构。该文件应具备两个用途：
- 供业务、产品、实施人员审阅
- 作为后续生成正式需求文档的高质量投喂材料

## 输出要求
- 优先记录事实，不写空话。
- 每个需求点必须带来源依据。
- 不确定内容单独列为 `待确认项`，不要混入正式结论。
- 若项目较大，可按功能模块拆分写。

## 简化规则

### 小型项目
- 若新增/优化/改造点数量较少，可合并成一个章节，但仍需保留三类标签。

### 大型项目
- 按模块分组输出：
  - 模块A-新增点
  - 模块A-优化点
  - 模块A-改造点

## 质量检查
- 每个需求点是否有来源依据
- 是否清楚区分新增/优化/改造
- 是否已写清影响范围
- 是否有待确认项
- 是否可直接作为正式需求文档输入材料
