# 用法示例

## 示例一：复刻企业级需求文档

### 用户输入

```text
请基于以下材料生成系统功能需求说明书，并输出中间稿、正式 markdown、正式 docx、自评报告：
- 会议纪要.docx
- 建设方案.docx
- 功能说明书.md
- 界面原型.pdf
- 公司需求模板.docx
```

### 期望执行
1. 读取全部材料并分类。
2. 抽取事实底稿，识别冲突与缺口。
3. 输出 `01-需求新增优化改造点.md`。
4. 融合公司模板与默认模板。
5. 输出 `02-系统功能需求说明书.md`。
6. 渲染 `02-系统功能需求说明书.docx`。
7. 输出 `03-AI自评报告.md`。

## 示例二：只有需求描述和截图

### 用户输入

```text
我要一份售后工单系统需求文档，现有材料只有一段需求描述和 5 张页面截图。
```

### 处理方式
- 先从需求描述和截图中抽取背景、目标、页面结构和关键操作。
- 对缺失的核心对象状态、角色权限、范围边界先提问。
- 先形成中间稿，再生成正式文档。

## 示例三：印章稽核类项目

### 常见材料组合
- 会议纪要
- 建设方案
- 功能说明书
- 页面 PDF 或截图
- 历史需求文档
- 公司模板

### 重点关注
- 是否存在标准库与业务流程的闭环
- 是否存在 AI 或异步能力
- 是否有"新增点/优化点/改造点"混杂表达
- 是否需要区分"范围内/范围外"

## 推荐文件命名

- `01-需求新增优化改造点.md`
- `02-系统功能需求说明书.md`
- `02-系统功能需求说明书.docx`
- `03-AI自评报告.md`

## 推荐脚本调用

### 提取 docx
```bash
python scripts/extract_docx.py "原材料.docx" --output "原材料提取.md"
```

### 渲染 docx
```bash
python scripts/render_requirement_doc.py "02-系统功能需求说明书.md" "02-系统功能需求说明书.docx"
```
