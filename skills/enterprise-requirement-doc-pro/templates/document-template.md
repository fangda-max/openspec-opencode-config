# 系统功能需求说明书模板

本模板用于生成 `02-系统功能需求说明书.md`。如用户提供公司模板，以公司模板章节为主；本模板作为默认补充结构。

## 1. 规模选择

| 规模 | 适用条件 | 推荐结构 |
|------|------|------|
| 轻量 | <=3 个功能模块，集成少 | 3 章 |
| 标准 | 4-10 个功能模块 | 7 章 |
| 完整 | 多系统集成、角色复杂、AI/异步能力明显 | 7 章 + 附录 |

## 4. 各章节写作要点

### 版本历史（必填）
- 位于文档开头（封面之后、正文之前）
- 每列含义：版本号、变更日期、作者、变更内容摘要
- 初始版本写"V1.0"或"V0.1"
- 每次重大修订递增版本号（V1.0 → V1.1 → V2.0）
- 频繁变更的需求建议在附录单独维护变更记录

### 一、背景与建设目标
- 说明业务背景、现状问题、建设必要性。
- 建设目标建议 3-5 条，尽量体现业务价值而非技术实现。
- 有内部概念或行业术语时补充术语表。

### 二、范围与边界
- 范围建议按"基础层/核心层/支撑层"分层描述。
- 边界必须同时写"包含"和"不包含"。
- 若存在外部依赖、线下流程、历史系统共存，也在本章说明。

### 三、总体业务流程概览
- 以业务语言描述主流程，不要直接写接口流程。
- 每个流程至少说明"触发起点、核心环节、终点"。
- 如存在数据回流、反哺、闭环，要单独写一节。

### 四、系统功能
- 这是正文核心。
- 简单模块可写"业务场景与流程 + 功能详情"。
- 复杂模块必须写"业务目标 + 业务场景 + 业务流程 + 页面/操作详情"。
- 页面/操作详情统一使用下方模板。

### 五、非功能需求
- 默认至少覆盖性能、安全、审计三类。
- 缺少具体指标时，可使用企业级通用占位值，但要注明"建议值"或"待确认"。

### 六、系统接口
- 无集成可省略。
- 有集成但信息不完整时，先列接口清单并标"待研发梳理"。

### 七、权限管理
- 角色复杂时单列。
- 若权限简单，可合并入各功能模块，但至少在本章给出角色矩阵摘要。

## 5. 功能模块详情模板（Pro 版增强）

### A 类功能模板

```markdown
##### X.X.X 页面/操作名称

【操作入口】：菜单路径、父页面按钮或流程节点

【功能描述】：一句话说明该功能解决的问题

【使用角色】：角色A、角色B

【界面原型】：
- 查询区：字段、筛选条件、按钮
- 列表区：表头、操作列、状态展示
- 表单区：分组、字段、上传区、说明区
- 底部操作区：提交、保存、关闭、返回

【AC 验收标准】（Pro 版新增：必须采用 Given-When-Then 格式，至少 3 个）

| 场景 | Given（前置条件） | When（用户操作） | Then（系统响应） |
|------|------------------|-----------------|----------------|
| 主流程-正常创建 | 用户已登录，有权限，必填字段已填写 | 点击【提交】按钮 | 订单创建成功，状态="待审核"，跳转列表页，显示"创建成功"提示 |
| 异常流-字段缺失 | 用户已登录 | 必填字段为空，点击【提交】 | 提示"XX字段不能为空"，高亮错误字段，不提交，停留在当前页 |
| 异常流-权限不足 | 用户已登录，无权限 | 点击【提交】按钮 | 提示"您无权限执行此操作"，按钮置灰或隐藏 |
| 边界流-并发冲突 | 用户A和用户B同时编辑同一条数据 | 用户A先点击【保存】 | 用户B保存时提示"数据已被修改，请刷新后重试"，数据不覆盖 |

【Unhappy Path 详细说明】（Pro 版新增：至少 2 个）

**UP1：业务异常 - 数据不存在**
- 触发条件：用户点击【编辑】，但数据已被删除
- 系统响应：返回列表页，提示"数据不存在或已被删除"
- 错误代码：DATA_NOT_FOUND
- 用户提示："抱歉，该数据不存在或已被删除，请刷新列表"

**UP2：系统异常 - 网络超时**
- 触发条件：提交时网络中断或服务器响应超时（>30s）
- 系统响应：提示超时，数据状态不变，允许重试
- 错误代码：TIMEOUT_ERROR
- 用户提示："网络连接超时，请检查网络后重试"
- 降级策略：本地缓存数据，网络恢复后自动重试

【实现逻辑】：
- 页面初始化：进入页面时系统默认执行什么
- 查询逻辑：条件组装、校验、默认排序、分页
- 按钮逻辑：按钮权限、点击动作、校验规则、处理结果
- 状态流转：状态A -> 触发条件 -> 状态B
- 异常处理：校验失败、权限不足、无数据、执行失败时的反馈

【字段说明】：
| 字段名 | 类型 | 长度 | 必填 | 可编辑 | 默认值 | 说明 |
|------|------|------|------|------|------|------|
| XXX | 文本 | 50 | 是 | 否 | - | XXX |

【数据追踪埋点】（Pro 版新增：核心功能必须填写）

| 事件名称 | 触发时机 | 关键属性 | 用途 |
|---------|---------|---------|------|
| 页面访问 | 进入页面时 | user_id, page_name, source | 功能使用率分析 |
| 按钮点击 | 点击【提交】时 | user_id, button_name, form_data_hash | 转化率分析 |
| 操作成功 | 操作成功完成 | user_id, operation_type, duration_ms | 成功率、性能监控 |
| 操作失败 | 操作失败时 | user_id, error_code, error_msg | 错误率分析 |

【兼容性分析】（Pro 版新增：仅改造点需要）
- 历史数据迁移：是否需要迁移旧数据？迁移方案是什么？
- 接口兼容性：接口是否有变化？是否向后兼容？
- 功能回滚：如果新版本出现问题，能否回滚到旧版本？
- 灰度策略：是否支持灰度发布？灰度范围如何控制？
```

## 7. 数据追踪埋点章节详解（Pro 版新增）

### 8.1 埋点事件列表

```markdown
| 事件名称 | 事件类型 | 触发页面/操作 | 触发时机 | 业务含义 |
|---------|---------|--------------|---------|---------|
| xxx_page_view | 页面浏览 | XXX页面 | 页面加载完成 | 统计功能使用率 |
| xxx_button_click | 按钮点击 | XXX页面-提交按钮 | 用户点击按钮 | 统计操作触发率 |
| xxx_operation_success | 操作成功 | XXX页面 | 操作成功完成 | 统计成功率和耗时 |
| xxx_operation_fail | 操作失败 | XXX页面 | 操作失败时 | 统计错误率和错误类型 |
```

### 8.2 事件属性定义

```markdown
【通用属性（所有事件都包含）】
| 属性名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| user_id | String | 用户唯一标识 | "U123456" |
| session_id | String | 会话标识 | "S789012" |
| timestamp | Long | 事件时间戳（毫秒） | 1712054400000 |
| device_type | String | 设备类型 | "PC" / "Mobile" |

【事件特有属性】
| 事件名称 | 属性名 | 类型 | 说明 | 示例 |
|---------|-------|------|------|------|
| xxx_page_view | page_name | String | 页面名称 | "订单创建页" |
| xxx_page_view | source_page | String | 来源页面 | "订单列表页" |
| xxx_button_click | button_name | String | 按钮名称 | "提交订单" |
| xxx_button_click | form_data_hash | String | 表单数据摘要（脱敏） | "a1b2c3..." |
| xxx_operation_success | duration_ms | Long | 操作耗时（毫秒） | 1500 |
| xxx_operation_fail | error_code | String | 错误代码 | "VALIDATION_ERROR" |
```

### 8.3 埋点实施要求

- 所有核心操作必须埋点（页面访问、按钮点击、操作成功/失败）
- 敏感数据必须脱敏处理（手机号、身份证号、金额等）
- 埋点数据保留期限：≥6个月
- 埋点实时性要求：T+1 可查询
- 埋点准确性要求：丢失率 < 1%

## 8. 兼容性分析章节详解（Pro 版新增）

### 适用场景
- 系统改造项目（非全新建设）
- 涉及历史数据迁移
- 接口变更或替换
- 功能逻辑重大调整

### 分析维度

```markdown
【历史数据迁移】
- 旧数据量：预估需要迁移的数据量（条数/大小）
- 迁移方案：全量迁移/增量迁移/双写机制
- 迁移时间：预计迁移时长，是否需要停机维护
- 数据清洗：是否需要数据清洗或格式转换
- 回滚方案：迁移失败时的回滚策略

【接口兼容性】
- 接口变更类型：新增/修改/删除/废弃
- 向后兼容：新版本是否兼容旧接口调用方
- 调用方影响：影响哪些下游系统？需要对方配合改造吗？
- 切换策略：一刀切/灰度/双轨并行
- 废弃接口下线时间：旧接口何时彻底下线？

【功能回滚】
- 回滚触发条件：什么情况下需要回滚？
- 回滚范围：全量回滚/部分回滚/数据回滚
- 回滚时间窗口：回滚需要在多长时间内完成？
- 回滚验证：回滚后如何验证系统正常？

【灰度发布策略】
- 灰度范围：按用户/按部门/按地区/按流量比例
- 灰度时长：灰度观察期多长时间？
- 灰度指标：什么指标达标可以全量发布？
- 紧急回滚：灰度期间发现问题如何紧急回滚？
```
